# Yarn Berry Lockfile Support Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Resolve exact installed versions from Yarn Berry (v2+) lockfiles, closing the gap left by the `v1.3.0` lockfile-resolution feature, which only supported npm, Yarn Classic, and pnpm.

**Architecture:** A new `src/lockfiles/yarnBerry.ts` parser, structurally identical to the existing Yarn Classic parser but adjusted for Berry's YAML-style version syntax and protocol-prefixed descriptors. The dispatcher (`src/lockfiles/index.ts`) content-sniffs `yarn.lock` for a `__metadata:` block (present only in Berry lockfiles) to decide which parser to use — one file read either way.

**Tech Stack:** TypeScript, Vitest. No new dependencies.

## Global Constraints

- Yarn Classic (`src/lockfiles/yarn.ts`) is already shipped and approved — do not modify it. The new Berry parser is a separate sibling file.
- Only `npm:`-protocol descriptors are matched (e.g. `lodash@npm:^4.17.21`). `workspace:`/`patch:`/`file:`-protocol descriptors are not real registry packages — a name whose range corresponds to one of these falls back to unversioned, same as any other unresolvable name. Never a hard failure.
- Detection: `yarn.lock` containing a `__metadata:` line → Berry parser. Otherwise → existing Classic parser, unchanged behavior.
- PnP mode requires no special handling — it changes how `node_modules` is populated, not the `yarn.lock` format, so parsing the lockfile alone supports both linker modes.

**Design notes for the implementer:**

Berry's multi-descriptor lines look different from Classic's at first glance — Classic quotes each descriptor separately: `"lodash@^4.17.15", "lodash@^4.17.21":`. Berry wraps the *entire* comma-separated list in one pair of quotes: `"lodash@npm:^4.17.15, lodash@npm:^4.17.21":`. Despite this, the same descriptor-parsing logic works unchanged for both: strip the trailing `:`, split on `, `, then strip a leading/trailing `"` from each piece. Splitting `"lodash@npm:^4.17.15, lodash@npm:^4.17.21"` on `, ` yields `['"lodash@npm:^4.17.15', 'lodash@npm:^4.17.21"']` — the first piece keeps its leading quote, the last keeps its trailing quote, and stripping `^"|"$` from each cleans both correctly. Verify this yourself by tracing it before you write the fixture, don't just trust this description.

- [ ] **Step 3: Implement `src/lockfiles/yarnBerry.ts`**

```typescript
import type { NamedRange } from './types'

function parseDescriptors(headerLine: string): string[] {
  return headerLine
    .replace(/:$/, '')
    .split(', ')
    .map(d => d.trim().replace(/^"|"$/g, ''))
}

export function resolveYarnBerryVersions(
  lockfileContent: string,
  packages: NamedRange[],
): Map<string, string> {
  const descriptorToVersion = new Map<string, string>()
  const lines = lockfileContent.split('\n')

  let currentDescriptors: string[] = []
  for (const line of lines) {
    if (line.startsWith('#') || line.trim() === '') continue

    if (!line.startsWith(' ')) {
      const descriptors = parseDescriptors(line)
      // __metadata is Berry's lockfile-format header block, not a package —
      // skip it so its own `version:` line is never captured as a resolution.
      currentDescriptors = descriptors[0] === '__metadata' ? [] : descriptors
      continue
    }

    const versionMatch = line.match(/^\s*version:\s*(\S+)/)
    if (versionMatch && currentDescriptors.length > 0) {
      for (const descriptor of currentDescriptors) {
        descriptorToVersion.set(descriptor, versionMatch[1]!)
      }
      currentDescriptors = []
    }
  }

  const resolved = new Map<string, string>()
  for (const { name, range } of packages) {
    if (!range) continue
    const version = descriptorToVersion.get(`${name}@npm:${range}`)
    if (version) resolved.set(name, version)
  }

  return resolved
}
```

Note: `parseDescriptors` is intentionally duplicated from `src/lockfiles/yarn.ts` rather than shared, keeping the two format parsers fully independent (per the design decision to leave the already-shipped Classic parser untouched). The two differences from the Classic parser are: the version-line regex (`version:\s*(\S+)` instead of `version\s+"([^"]+)"`) and the descriptor lookup key (`${name}@npm:${range}` instead of `${name}@${range}`).

import * as fs from 'fs'
import * as path from 'path'
import { resolveNpmVersions } from './npm'

import { resolvePnpmVersions } from './pnpm'
import type { NamedRange } from './types'

export function resolveLockfileVersions(
  lockfileDir: string,
  packages: NamedRange[],
): Map<string, string> {
  const npmPath = path.join(lockfileDir, 'package-lock.json')
  if (fs.existsSync(npmPath)) {
    return resolveNpmVersions(fs.readFileSync(npmPath, 'utf8'), packages)

  if (fs.existsSync(yarnPath)) {

  }

  return new Map()
}

```typescript
import * as fs from 'fs'
import * as path from 'path'
import { resolveNpmVersions } from './npm'
import { resolveYarnVersions } from './yarn'
import { resolveYarnBerryVersions } from './yarnBerry'
import { resolvePnpmVersions } from './pnpm'
import type { NamedRange } from './types'

export type { NamedRange } from './types'

export function resolveLockfileVersions(
  lockfileDir: string,
  packages: NamedRange[],
): Map<string, string> {
  const npmPath = path.join(lockfileDir, 'package-lock.json')
  if (fs.existsSync(npmPath)) {
    return resolveNpmVersions(fs.readFileSync(npmPath, 'utf8'), packages)
  }

  const yarnPath = path.join(lockfileDir, 'yarn.lock')
  if (fs.existsSync(yarnPath)) {
    const yarnContent = fs.readFileSync(yarnPath, 'utf8')
    return yarnContent.includes('__metadata:')
      ? resolveYarnBerryVersions(yarnContent, packages)
      : resolveYarnVersions(yarnContent, packages)
  }

  const pnpmPath = path.join(lockfileDir, 'pnpm-lock.yaml')
  if (fs.existsSync(pnpmPath)) {
    return resolvePnpmVersions(fs.readFileSync(pnpmPath, 'utf8'), packages)
  }

  return new Map()
}
```
